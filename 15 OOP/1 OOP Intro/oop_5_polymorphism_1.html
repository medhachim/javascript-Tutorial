OOP: Polymorphism


  • Example:
    A button has a function to press.
    When no 1 is pressed the output is 'balance'
    When no 2 is pressed the output is 'data'
    When no 3 is pressed the output is 'validity'
    When no 4 is pressed the output is 'customer care'
    Here the function is same 'press button' but the input and output are different.
    The function chooses the block of statment to be executed for the output according to the input given. When we press 1 or 3, the same button function is called but executing different block of statment for which input it is framed. So, it is possible because of the polymorphism concept.
