IIFE(Immediately Invoked Function Expression)

It is a JavaScript Function that runs as soon as it is defined. It is not needed to call. It is called by itself.
It is a design pattern which is also known as Self-Executing Anonymous Function and contains two major parts.
The first is anonymous with lexical scope enclosed within the Grouping Operator(). This prevents accessing variables within IIFE idiom as well as polluting the global scope.
The second part is creating the immediately executing functin expression(), through which the JavaScript engine will directly interprete the function.
    Points to remember.
  • Avoid creating global variable and functions. It's a bad practice to have plenty of global variable.
  • As it doesn't define a variable and function globally, there will be no name conflict.
  • Scope is limited to that particular function. A variable created inside IIFE function, it's scope is limited to that IIFE function only. It cannot be accessed from outside.
The first part of IIFE function is an anonymous function with grouping operators.
 Syntax:
//(first part)(second part);
eg:(function(){document.write("Rose Tamang");})();
passing parameter in the second part
eg:(functin(a,b){document.write(a+ " "+b);})(10,20);
