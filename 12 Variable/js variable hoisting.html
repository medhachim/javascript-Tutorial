Variable Hoisting

Hoisting is Javascript's default behaviour of moving declaration to the top of the function, if defined in a function, or the top of the global context, if defined outside a function.
Javascript divides the code var a = 10 in to two parts.
var a; Variable declaration: Javascript moves this declaration to the top if it is inside the function or to the top of the global context if it is outside the function
a = 10; Variable initialization.
  • We write like this:
    var a = 10;
    document.write(a);
    var b = 20;
  • Compile Phase:
    var a; //The declaration is moved to the top.
    var b; //The declaration is moved to the top
    a = 10; //Initialization remains where it was.
    document.write(a); //Code remains where it was.
    b = 20; //Initialization remains where it was.
  • Javascript uderstands the following code:
    a = 10;
    document.write(a);
    var a;
    as:
    var a;
    a = 10;
    document.write(a);
    So it makes no difference where you declare the variable. But initialization follows the precedence.
  • A variable can be used before it has been declared.
  • A variable can be used before it has been declared.
